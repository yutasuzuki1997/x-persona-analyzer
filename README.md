# X Comment Persona Analyzer

XのポストへのリプライをAIで分析し、**賛同者・批判者のペルソナ**を可視化するWebアプリです。

---

## できること

- 指定したXのポストURLからリプライを最大500件取得（ページネーション対応）
- Claude AIがリプライ全件を分析し、以下を出力
  - 🤖 AIっぽいアカウントの割合・根拠・アカウント名
  - 👍 賛同者の典型像（件数・割合・キーワード・代表投稿）
  - 👎 批判者の典型像（件数・割合・キーワード・代表投稿）
  - 💡 全体から得られるインサイト
- 分析結果をXに投稿するためのテキストをワンクリックでコピー

---

## システム構成

```
ブラウザ (Vite + React)
    │
    │  HTTP (localhost:3001)
    ▼
Node.js プロキシ (x-proxy.js)
    ├── GET /replies      → X API (tweets/search/recent)
    ├── GET /user-context → X API (users/:id/tweets)
    └── POST /claude      → Anthropic API (claude-sonnet-4-6)
```

**なぜプロキシが必要か**
ブラウザから X API・Anthropic API を直接呼ぶと CORS でブロックされるため、ローカルの Node.js サーバーを中継しています。APIキーもブラウザには渡さず、プロキシ経由でのみ使用します。

## セットアップ

### 必要なもの

- Node.js v18 以上
- X Developer アカウントの Bearer Token（Basic プラン以上推奨）
- Anthropic API Key

### 手順

```bash
# 1. 依存パッケージをインストール
npm install

# 2. ターミナル1：プロキシサーバーを起動
node x-proxy.js
# → ✅ X API Proxy起動中: http://localhost:3001 が出ればOK

# 3. ターミナル2：フロントエンドを起動
npm run dev
# → http://localhost:5173 をブラウザで開く
```

## 使い方

### 1. Bearer Token を取得する

1. [developer.x.com](https://developer.x.com) にアクセスしてログイン
2. 「Projects & Apps」から新規アプリを作成
3. 「Keys and Tokens」タブを開く
4. 「Bearer Token」を生成・コピー

### 2. 初期設定

- ブラウザで `http://localhost:5173` を開く
- X Bearer Token と Claude API Key を入力して「設定を保存して次へ」
- 設定は `localStorage` に保存されるため、次回以降は入力不要
- 後から変更したい場合は「設定を変更」リンクから戻れる

### 3. 分析する

- 分析したいXのポストURLを入力
- 「取得→分析する」ボタンをクリック
- リプライ取得 → ユーザー情報収集 → AI分析 の順で自動実行（進行ログ表示）

### 4. 結果を確認・シェア

- 分析カードで賛否・AIアカウント割合・代表投稿・インサイトを確認
- 「コピー」ボタンでX投稿用テキストをクリップボードにコピー

## AIっぽいアカウントの判定ロジック

AIアカウントの判定は **Claude AIがリプライ内容・ユーザー情報をもとに総合判断** します。コードからClaudeに渡している判定材料は以下の通りです。

| 材料 | 内容 |
|---|---|
| 投稿パターン | リプライ本文（全件） |
| フォロワー情報 | `followers_count` / `following_count` の比率 |
| プロフィール文 | `description`（自己紹介） |
| 最近の投稿 | 各ユーザーの直近10件のツイート本文 |

Claudeへのプロンプトでは「botらしい投稿パターン・フォロワー比率など」を根拠として挙げるよう指示しており、疑わしいユーザー名のリストと割合（0〜100の整数）・説明文をJSON形式で返させています。ルールベースの自動判定ではなく、文脈を読んだLLMによる推定です。

## 取得できるデータの仕様

| 項目 | 仕様 |
|---|---|
| 取得上限 | 最大500件（100件/ページ × ページネーション） |
| 対象期間 | **直近7日以内**のリプライのみ（X APIの`search/recent`の制限） |
| ユーザーコンテキスト | リプライしたユーザー全員の直近10件の投稿を取得 |
| リクエストボディ上限 | 10MB（Expressの設定値） |

> **7日制限について**：`tweets/search/recent` エンドポイントは過去7日間のツイートしか検索できません。7日以上前のポストを指定した場合、リプライ0件になります。

## 注意事項

### X API の料金について

- **Free プラン**：`conversation_id` による検索（リプライ取得）は利用不可の場合があります
- **Basic プラン（$100/月）**：月100万件の読み取りが可能。このツールの通常利用には十分です
- ユーザーの投稿履歴取得（`/user-context`）はリプライ件数分のAPIリクエストが発生します。大量のリプライがあるポストでは消費量に注意してください

### Bearer Token の取り扱い

- Bearer Token と Claude API Key はブラウザの `localStorage` に保存されます
- コードにはハードコードされていません
- このツールはローカル専用のため、トークンがサーバーに送信されることはありません
- ただし `localStorage` はブラウザの開発者ツールから参照可能です。共有PCでの利用は避けてください
